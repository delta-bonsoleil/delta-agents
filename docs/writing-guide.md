# エージェント定義の書き方

## ファイル形式

エージェント定義は `.md` ファイルで、frontmatter + 本文の構成をとる。

## Frontmatter（必須）

```yaml
---
name: エージェント名
description: 一行の説明（Claude Codeのエージェント選択時に表示される）
tools: 使用可能なツール（カンマ区切り）
model: 使用モデル（opus, sonnet, haiku）
---
```

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `name` | Yes | エージェントの識別名。英数字小文字推奨 |
| `description` | Yes | 一行の説明。エージェント一覧での表示に使われる |
| `tools` | Yes | エージェントが利用できるツール。例: `Read, Grep, Glob, Bash, WebSearch, WebFetch` |
| `model` | No | 使用するモデル。省略時は親のモデルを継承 |

## 本文の構成

frontmatterの後にエージェントのプロンプト本文を記述する。以下のセクションを推奨：

### 1. 冒頭の一文
エージェントが「自分は何者か」を認識するための宣言。

### 2. プロフィール
名前、外見、役職など、キャラクターとしての基本情報。

### 3. 性格
振る舞いの方向性。どんなトーンで応答するか。

### 4. 役割
エージェントが担当する業務領域の一覧。

### 5. 作業手順
タスクを遂行する際の標準的なステップ。

### 6. 行動規範
エージェントが従うべきルールや価値観。

### 7. 口調
一人称、語尾、特徴的なフレーズなど具体的な話し方の例。

### 8. 専門領域での姿勢（任意）
特定の業務に対するスタンスやポリシー。

## 配置

所属する部署のディレクトリに配置する。ファイル名はエージェント名と一致させる。

```
agents/{部署名}/{エージェント名}.md
```
